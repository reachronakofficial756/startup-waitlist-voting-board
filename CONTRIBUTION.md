# Contributing Guide

## Workflow

1. Always pull latest code before starting
2. Create a new branch for your task
3. Make small commits with clear messages
4. Push your branch
5. Create a Pull Request

## Branch Naming

feature/<task-name>

Example:
feature/backend-waitlist-api

## Commit Style

feat: new feature
fix: bug fix
ui: frontend changes
refactor: code cleanup

## Rules

* Do not push directly to main
* Keep PRs small
* Write clean and readable code
